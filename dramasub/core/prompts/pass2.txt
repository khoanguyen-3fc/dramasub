## Rules
- Natural spoken ${target_language}; meaning and tone over word-for-word.
- Keep exactly: speaker, addressee, grammatical person, negation,
  question-vs-statement. A character who calls themself by name keeps that.
- Pronouns and speech level follow the address table; for uncovered pairs infer
  from age, rank, gender, intimacy — modern register only unless the drama is
  historical. Each character above is tagged [nam] (male) or [nữ] (female): never
  use a female pronoun (cô/chị/bà) for a [nam] character, nor a male one (anh/ông)
  for a [nữ]. Commit to ONE term per line — never hedge with a slashed pair
  ("anh/cô", "em/tôi"); if unsure, choose what the gender tags and address table
  support. A speaker's self-term follows their OWN standing, not the listener's:
  a senior speaking to a junior stays "tôi" (or their title), never "em"/"tớ" —
  but a junior speaking up to a senior still uses the deferential "em".
- Job titles and ranks by workplace meaning, never by a similar-sounding word.
  When someone is addressed or named by rank (책임님, 선임님, 사장님), use their
  target-language title or name, not a bare pronoun that could mis-state gender.
- Interjections by function (a call stays a call), never escalated.
- Idioms and slang: the common ${target_language} equivalent, or plain wording
  any viewer understands. Write ${target_language} only — never another
  language's words or script.

## Output
Return ONE JSON object, nothing else: the same indices as keys (strings), each
mapped to its translation.
- Every index above, no others; no empty values; content stays with its own
  line even when a sentence spans cues.
- Keep ``{...}`` tags verbatim.
- Each value is a single line, no ``\N`` — the tool wraps; keep it readable
  within two screen lines of about ${max_line_chars} characters.
